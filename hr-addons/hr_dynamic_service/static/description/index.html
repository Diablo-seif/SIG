HR Dynamic Approval

Overview

Provide the ability to create a workflow for different types of requests and set access rights for each step in the workflow, every request could have multiple subtypes requests following the same flow of the main request.

Requested users can view the requests in a dashboard grouped by their type and have overview on the requests status.

Features

  • Create dynamic workflow.
  • Decide who will approve every step of the request.
  • Set action that will be triggered with a specific approval.
  • Users can view his requests in the dashboard.

Configuration

Create dynamic approval type The dynamic approval type has 3 main sections: (Approval type basic information, Approval subtypes if there are any, Approval stages.)

First, the user will give the approval a name, choose its icon and this approval type sequence between other types.

Second, if the approval have different subtypes the user can set multiple subtypes for it

Third, in the stages section the user can configure the request workflow and assign a user or group of users to approve every stage.

  • The “Name” represents the request stage and “Action Name” is the button name that will change the request stage to this stage, the user can decide if the request in this stage can be edited or be refused.
  • For the stage access rights the user has multiple options to decide who will approve this stage he can choose (Odoo Group, Multiple users, Employee Manager, Department Head, or the employee itself).
  • The user can choose server action that will be triggered with this stage approval, this action could be a record that will be created in another model.

How to Work

Create approval request: Once the user chooses the request type its stages will automatically appear on the request form.

After saving the request the button for the next stage will appear and in the “More Details” section a list of the user(s) that can click on the button will appear to the user.

Approval requests dashboard: Here the user can view his approval requests with their subtypes, and number of requests in every subtype and their stage.

Notes:This module is tested and works on Odoo Community and Enterprise Edition.
